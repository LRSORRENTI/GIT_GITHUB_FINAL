Writing custom git aliases

There's two things we'll go over:
  • How we write custom aliases.
  • The global config file (ie where we store them)
Some folks love using aliases since they're like shortcuts, custom commands you can write, so you don't need to type out full lengthy git commands

But before we dive in to writing git aliases, let's quickly go over the global config file. Git looks for the global config file at either ~/.gitconfig or ~/.config/git/config

Any configuration variables we change in that file will also be applied across all of our Git repos, but we can also alter configuration variables from the command line if preferred

We talked a little bit about the config file, when we used the 'ls .git' command, and the wide array of files and folders popped up, including that 'objects' folder from our: Inside Git: The OBJECTS Folder, the section where we took a look at how git is set up behind the scenes

But we can also set the global git config, and there's actually one level above that called 'system-wide git config file' so if you had a few different users all using the same machine, you could set the system-wide git config file so all the users config files were pre-set

But if we just use one account per machine, the global config file works just fine, remember at the beginning of the course we used the command: git config --global user.name, and user.email

But we can also do this by opening the hidden global git config file, we can print it out with: cat ~/.gitconfig

$ cat ~/.gitconfig [filter "lfs"] smudge = git-lfs smudge -- %f process = git-lfs filter-process required = true clean = git-lfs clean -- %f [user] name = Luke Sorrenti email = lrsorrenti@hotmail.com [core] editor = code --wait

We can open that gitconfig folder and edit it in VSCODE with the command: code ~/.gitconfig, if we wanted to change our settings in VSCODE instead of the command line

Writing our first alias

We can easily set up git aliases to make things simpler and faster, for example, we could define an alias 'git ci' instead of having to write git commit, or we can define a custom alias that will print a custom, formmatted git log with an alias like 'git lg'

We can set this up in our gitconfig file in VSCODE by adding: [alias] s = status, after saving, everytime we type 'git s' the status log will display, we created a shortcut, an alias

We could also do this in the command line itself with the command: git config --global alias.s status, the same thing will be achieved as above where we configured our 's' alias to display status as a shortcut, 'git s' is the same as 'git status'

Aliases with arguments

For example, we could set 'git add' to be 'git a', followed by two arguments. [alias] a = add file1 file2 file3, so we could do 'git a README.md index.html'

Aliases are automatically passed through after you call it

Exploring existing and useful git aliases online

There are tons of custom git aliases we can search for and use, pre-built aliases we can copy paste into our command line, aliases that can update the display of git log, aliases that make it easier to list all user configurations, $ git config --global alias.gl 'config --global -l', now 'git gl' displays all defined aliases and other configuration options

Also added [alias] ll = log --oneline Which shortens git log --oneline
